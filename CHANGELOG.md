# Changelog

hajimu_gui の全バージョン履歴

---

## v14.2.1 (2026-07-14) — キャンバス文字の座標修正

- キャンバス内の文字だけが平行移動・拡大縮小を無視する問題を修正
- 入れ子の子ウィンドウ内で文字が画面左上へずれ、クリップされる問題を修正
- 子ウィンドウ内キャンバスの回帰確認用サンプルを追加

---

## v14.2.0 (2026-07-14) — 実行環境パスの取得

- 現在プラグインを読み込んでいるHajimu本体の絶対パスを返す `実行ファイルパス` / `executable_path` を追加
- GUIアプリから同じHajimuバージョンの子プロセスを確実に起動可能に変更

---

## v14.1.1 (2026-07-14) — プロセス起動失敗の検出

- macOS／Linuxで外部実行ファイルが存在しない場合も成功扱いになる問題を修正
- `close-on-exec`パイプで、切り離したプロセスの`exec`成否を呼び出し元へ通知

---

## v14.1.0 (2026-07-14) — 安全な外部プロセス起動

- 実行ファイルと引数配列を直接OSへ渡す `外部プロセス起動` / `process_launch` を追加
- シェル文字列を組み立てないため、パスや引数を介したコマンド注入を防止
- macOS／Linuxでは親GUIから切り離し、Windowsではプロセスハンドルを直ちに解放

---

## v14.0.4 (2026-07-13) — 編集画面向けレイアウト修正

- `ウィジェット幅`の指定を次の入力欄やリストへ正しく反映
- `タブバー`の選択値に数値インデックスと文字列ラベルの両方を使用可能に変更
- `スクロール領域開始`の幅・高さに0以下を渡した場合、親領域の残り寸法を使用
- スクリーンショットを描画完了後、バッファ交換前に取得して欠落を防止

---

## v14.0.3 (2026-07-13) — 横並びレイアウトの安定化

- `横並び開始` / `横並び終了` で文字、入力欄、ボタンを正しく横方向へ配置
- 横並びの子ウィンドウ内では縦方向へ戻し、入れ子の横並びと干渉しないよう修正
- 横並び終了後の要素が操作列や子ウィンドウへ重なる問題を修正
- `折返しテキスト`、`色付きテキスト`、`無効テキスト`の矩形記録を修正
- `ウィジェット矩形`が基本ウィジェットの最新座標を返すよう修正
- `フレーム数`が描画ループごとに増加しない問題を修正
- 通常ウィジェット、入れ子、子ウィンドウの非交差を確認する回帰テストを追加

---

## v14.0.2 (2026-07-13) — 安全なファイル整理

- 通常ファイルだけを削除する `ファイル削除` / `file_delete` を追加
- ディレクトリを削除対象から除外し、素材管理からの誤操作を防止

---

## v14.0.1 (2026-07-13) — Hajimu v1.5 対応

- プラグインABIヘッダーを Hajimu v1.5.0 と同期
- 画像や音声を壊さずコピーできる `ファイルコピー` / `file_copy` を追加
- パッケージメタデータのバージョンを実装と一致させた

---

## v14.0.0 (2026-02-22) — Phase 131-134

**1,101関数 / 23,000行超 / 134 Phases**

7つのアーキテクチャ改善を新規 C ファイル 9枚で実装。

### Phase 131: 宣言的UI / 仮想UIツリー (v14.0.0)
- **VNodeツリー**: ダブルバッファプールアロケータ (4096ノード × 2面)
- **O(n)差分 Reconciler**: keyベースの照合で再描画ノードだけ更新
- **Flexboxレイアウト**: flex-grow / flex-dir / gap / padding 対応
- **トランジションアニメーション**: transition_ms で座標補間
- **ダーティ矩形管理**: AABBマージによる副作用决定領域の最小化
- 新規 API 12関数: `VUI.開始` `VUI.終了` `VUI.ボックス` `VUI.テキスト` `VUI.ボタン` `VUI.行` `VUI.列` `VUI.スクロール` `VUI.入力` `VUI.スタイル` `VUI.コミット` `VUI.無効化`

### Phase 132: フレームスケジューラ / GPUパイプライン最適化 (v14.0.0)
- **VSync 制御**: `hjp_gl_set_swap_interval(1)` によるドライバレベル同期
- **アダプティブスロットリング**: アイドルフレームで CPU 絶やす
- **描画コールキュー**: 4096 コールのバッチ (Rect/RoundRect/Text/Circle/Line)
- **FPS 計測**: 128フレームをローリング平均
- 新規 API 5関数: `フレーム設定` `フレームレート取得` `フレーム時間` `フレーム要求` `描画コール数`

### Phase 133: ホットリロード (v14.0.0)
- **stat() mtime ポーリング**: kqueue/inotify 不要 — クロスプラットフォーム対応
- **最大32ファイルを同時監視**、ポーリング間隔設定可
- **UIステートのスナップショット/復元** API
- 新規 API 5関数: `ホットリロード開始` `ホットリロード監視` `ホットリロード確認` `ホットリロードリセット` `ホットリロード回数`

### Phase 134: DevTools オーバーレイ (v14.0.0)
- **F12 トグル**: hjp_render で描画するインスペクタパネル
- **FPS グラフ**: 128フレーム履歴の折れ線グラフ
- **ウィジェットインスペクタ**: マウスホバーでノード情報 (key/型/座標/サイズ/hash) を表示
- **ログコンソール**: 32行リングバッファの画面内ログ
- **パフォーマンス**: CPU時間/描画ノード数/スキップ数 をリアルタイム表示
- 新規 API 5関数: `DevTools開始` `DevTools表示` `DevTools非表示` `DevToolsログ` `DevToolsオプション`
