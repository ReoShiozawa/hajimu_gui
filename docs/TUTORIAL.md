## 1. はじめに

### 前提条件

- はじむ言語（nihongo）がインストール済みであること
- hajimu_gui がビルド・インストール済みであること

```bash
# インストール
git clone https://github.com/ReoShiozawa/hajimu_gui.git
cd hajimu_gui
make
make install
```

### hajimu_gui の特徴

- **日本語で書ける** — すべての関数が日本語名で呼び出せます
- **外部依存なし** — OS 標準 API のみ使用（macOS: Cocoa、Linux: X11、Windows: Win32）
- **即時モード** — ボタンやテキストは毎フレーム描画関数を呼ぶだけ
- **1,074 の組み込み関数** — ウィジェット、描画、アニメーション、ファイル操作まで網羅

### 基本構造

hajimu_gui アプリケーションは、常にこの 4 ステップで構成されます：

```
① パッケージ読み込み
② アプリ作成（ウィンドウ生成）
③ 描画ループ（毎フレーム実行される関数）
④ アプリ終了
```

## 2. Hello World — 最初のウィンドウ

最小限のプログラムから始めましょう。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("はじめてのGUI", 400, 300)

GUI.描画ループ(アプリ, 関数():
    GUI.ラベル("こんにちは、世界！")
終わり)

GUI.アプリ終了(アプリ)
```

**実行：**

```bash
hajimu hello.jp
```

**コード解説：**

| 行 | 説明 |
|---|---|
| `取り込む "hajimu_gui" として GUI` | hajimu_gui パッケージを `GUI` という名前で読み込む |
| `GUI.アプリ作成("タイトル", 幅, 高さ)` | 指定サイズのウィンドウを作成 |
| `GUI.描画ループ(アプリ, 関数(): ... 終わり)` | メインループ。引数の関数が毎フレーム呼ばれる |
| `GUI.ラベル("テキスト")` | テキストを表示 |
| `GUI.アプリ終了(アプリ)` | ウィンドウを閉じてリソースを解放 |

> **💡 ポイント**: `描画ループ` の中の関数は毎フレーム呼ばれます。ウィジェットの状態は毎回関数を呼ぶことで画面に反映されます（即時モード）。

## 3. ラベルとテキスト表示

テキスト表示にはいくつかのバリエーションがあります。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("テキスト表示", 500, 400)

GUI.描画ループ(アプリ, 関数():
    GUI.ラベル("通常のテキスト")
    GUI.見出し("大きな見出し")
    GUI.小テキスト("小さなテキスト")
    GUI.区切り線()
    GUI.色付きラベル("赤いテキスト", "#FF0000")
    GUI.中央揃え()
    GUI.ラベル("中央に表示")
    GUI.右揃え()
    GUI.ラベル("右寄せ")
終わり)

GUI.アプリ終了(アプリ)
```

**主なテキスト関数：**

| 関数 | 説明 |
|---|---|
| `ラベル(テキスト)` | 通常テキスト |
| `見出し(テキスト)` | 大きな見出し |
| `小テキスト(テキスト)` | 小さめのテキスト |
| `色付きラベル(テキスト, 色)` | 指定色でテキスト表示 |
| `区切り線()` | 水平の区切り線 |
| `スペース(高さ)` | 縦方向の余白 |
| `中央揃え()` / `右揃え()` | テキスト配置変更 |

## 4. ボタンとイベント処理

ボタンはクリックされると `真` を返します。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("ボタン", 400, 300)
変数 回数 = 0

GUI.描画ループ(アプリ, 関数():
    GUI.ラベル("ボタンを押した回数: " + 回数)

    もし GUI.ボタン("押してね") なら
        回数 = 回数 + 1
    終わり

    もし GUI.ボタン("リセット") なら
        回数 = 0
    終わり
終わり)

GUI.アプリ終了(アプリ)
```

**コード解説：**

- `変数 回数 = 0` — 描画ループの**外**で状態変数を宣言する
- `もし GUI.ボタン("ラベル") なら ... 終わり` — ボタンが押されたときだけ中身が実行される
- 状態変数の更新は即座に画面に反映される（次フレームで再描画されるため）

> **💡 ポイント**: 即時モードでは「状態変数を描画ループの外で宣言」が基本パターンです。ループ内で宣言すると毎フレーム初期化されてしまいます。

**ボタンのバリエーション：**

```
GUI.ボタン("通常ボタン")
GUI.小ボタン("小さいボタン")
GUI.画像ボタン("icon.png", 32, 32)
GUI.トグルボタン("オン/オフ", 状態)
```

## 5. テキスト入力

テキスト入力は、現在の値を渡して更新された値を受け取ります。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("テキスト入力", 500, 400)
変数 名前 = ""
変数 パスワード = ""
変数 年齢 = 20
変数 自己紹介 = ""

GUI.描画ループ(アプリ, 関数():
    名前 = GUI.テキスト入力("名前", 名前)
    パスワード = GUI.パスワード入力("パスワード", パスワード)
    年齢 = GUI.数値入力("年齢", 年齢, 0, 120, 1)
    自己紹介 = GUI.テキストエリア("自己紹介", 自己紹介, 5)

    GUI.区切り線()
    GUI.ラベル("こんにちは、" + 名前 + " さん（" + 年齢 + "歳）")
終わり)

GUI.アプリ終了(アプリ)
```

**テキスト入力関数：**

| 関数 | 引数 | 説明 |
|---|---|---|
| `テキスト入力(ラベル, 値)` | ラベル, 現在値 | 一行テキスト入力。更新値を返す |
| `パスワード入力(ラベル, 値)` | ラベル, 現在値 | マスクされたテキスト入力 |
| `数値入力(ラベル, 値, 最小, 最大, ステップ)` | ラベル, 数値, 範囲, 刻み | 数値入力 |
| `テキストエリア(ラベル, 値, 行数)` | ラベル, 現在値, 行数 | 複数行テキスト入力 |
| `検索入力(ラベル, 値)` | ラベル, 現在値 | 検索アイコン付きテキスト入力 |

> **💡 ポイント**: テキスト入力は「渡した値を更新して返す」パターンです。必ず `変数 = GUI.テキスト入力(...)` のように戻り値を受け取ってください。

## 6. チェックボックスとスライダー

真偽値や数値の入力ウィジェットです。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("コントロール", 500, 400)
変数 有効 = 真
変数 音量 = 50
変数 色 = [128, 128, 255]
変数 選択 = 0

GUI.描画ループ(アプリ, 関数():
    有効 = GUI.チェックボックス("通知を有効にする", 有効)
    音量 = GUI.スライダー("音量", 音量, 0, 100)
    GUI.プログレスバー("進捗", 音量 / 100)
    色 = GUI.色選択("テーマ色", 色)
    選択 = GUI.ラジオボタン("サイズ", ["小", "中", "大"], 選択)

    GUI.区切り線()
    もし 有効 なら
        GUI.ラベル("通知: オン / 音量: " + 音量)
    でなければ
        GUI.ラベル("通知: オフ")
    終わり
終わり)

GUI.アプリ終了(アプリ)
```

**コントロール関数：**

| 関数 | 戻り値 | 説明 |
|---|---|---|
| `チェックボックス(ラベル, 値)` | 真偽 | オン/オフ切替 |
| `スライダー(ラベル, 値, 最小, 最大)` | 数値 | 範囲スライダー |
| `プログレスバー(ラベル, 値)` | — | 進捗表示（0.0〜1.0） |
| `色選択(ラベル, [R,G,B])` | 配列 | RGB カラーピッカー |
| `ラジオボタン(ラベル, 選択肢, 選択)` | 数値 | 排他選択 |
| `トグルスイッチ(ラベル, 値)` | 真偽 | iOS風スイッチ |

## 7. レイアウト — パネルと横並び

ウィジェットをグループ化して見やすく配置します。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("レイアウト", 600, 400)
変数 名前 = ""
変数 メール = ""

GUI.描画ループ(アプリ, 関数():
    // パネルでグループ化
    GUI.パネル開始("ユーザー情報")
        名前 = GUI.テキスト入力("名前", 名前)
        メール = GUI.テキスト入力("メール", メール)
    GUI.パネル終了()

    // 横並びでボタンを配置
    GUI.横並び開始()
        もし GUI.ボタン("送信") なら
            GUI.通知("送信しました！")
        終わり
        もし GUI.ボタン("クリア") なら
            名前 = ""
            メール = ""
        終わり
    GUI.横並び終了()

    // スクロール可能な領域
    GUI.スクロール領域開始("ログ", 200)
        繰り返す 変数 i = 1 から 50 まで
            GUI.ラベル("ログ行 " + i)
        終わり
    GUI.スクロール領域終了()
終わり)

GUI.アプリ終了(アプリ)
```

**レイアウト関数：**

| 関数ペア | 説明 |
|---|---|
| `パネル開始(名前)` / `パネル終了()` | 枠線付きグループ |
| `横並び開始()` / `横並び終了()` | 水平レイアウト |
| `スクロール領域開始(名前, 高さ)` / `スクロール領域終了()` | スクロール可能領域 |
| `折りたたみ開始(ラベル)` / `折りたたみ終了()` | 折りたたみ可能セクション |
| `インデント()` / `インデント解除()` | インデント |

> **💡 ポイント**: レイアウト関数は必ず `開始` と `終了` をペアで使います。ネストも可能です。

## 8. リストとコンボボックス

選択肢の一覧から値を選ぶウィジェットです。


## 9. ダイアログとメッセージ

ユーザーへの通知や確認に使います。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("ダイアログ", 500, 400)
変数 結果 = ""

GUI.描画ループ(アプリ, 関数():
    // 通知メッセージ（数秒で自動消滅）
    もし GUI.ボタン("通知を出す") なら
        GUI.通知("保存しました！")
    終わり

    // 確認ダイアログ（はい/いいえ）
    もし GUI.ボタン("確認ダイアログ") なら
        もし GUI.確認ダイアログ("本当に削除しますか？") なら
            結果 = "削除しました"
        でなければ
            結果 = "キャンセルしました"
        終わり
    終わり

    // メッセージダイアログ
    もし GUI.ボタン("メッセージ") なら
        GUI.メッセージダイアログ("お知らせ", "処理が完了しました。")
    終わり

    もし 結果 != "" なら
        GUI.ラベル(結果)
    終わり
終わり)

GUI.アプリ終了(アプリ)
```

**ダイアログ関数：**

| 関数 | 説明 |
|---|---|
| `通知(テキスト)` | 画面端に一時的な通知 |
| `確認ダイアログ(メッセージ)` | はい/いいえの確認。真偽を返す |
| `メッセージダイアログ(タイトル, メッセージ)` | OKボタン付きメッセージ |
| `入力ダイアログ(タイトル, 初期値)` | テキスト入力付きダイアログ |
| `ファイル選択ダイアログ(フィルタ)` | ファイル選択 |
| `保存ダイアログ(フィルタ)` | 保存先選択 |

## 10. タブで画面を整理する

複数の画面をタブで切り替えます。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("タブ", 600, 400)
変数 タブ = 0
変数 名前 = ""
変数 音量 = 50

GUI.描画ループ(アプリ, 関数():
    タブ = GUI.タブバー(["プロフィール", "設定", "ヘルプ"], タブ)

    もし GUI.タブ内容(タブ, 0) なら
        GUI.見出し("プロフィール")
        名前 = GUI.テキスト入力("名前", 名前)
    終わり

    もし GUI.タブ内容(タブ, 1) なら
        GUI.見出し("設定")
        音量 = GUI.スライダー("音量", 音量, 0, 100)
    終わり

    もし GUI.タブ内容(タブ, 2) なら
        GUI.見出し("ヘルプ")
        GUI.ラベル("hajimu_gui v13.0.0")
        GUI.ラベル("詳細は REFERENCE.md を参照してください。")
    終わり
終わり)

GUI.アプリ終了(アプリ)
```

**パターン：**

1. `タブバー(タブ名配列, 現在値)` でタブバーを描画し、選択中のインデックスを返す
2. `タブ内容(タブ値, インデックス)` で各タブの内容を条件分岐する

## 11. テーブルでデータを表示する

表形式のデータ表示には、テーブル関数を使います。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("テーブル", 600, 400)
変数 初期化済 = 偽
変数 テーブル = 無

GUI.描画ループ(アプリ, 関数():
    // テーブルは初回のみ作成
    もし 初期化済 == 偽 なら
        テーブル = GUI.テーブル作成(["名前", "年齢", "都市"])
        GUI.テーブル行追加(テーブル, ["田中太郎", "25", "東京"])
        GUI.テーブル行追加(テーブル, ["佐藤花子", "30", "大阪"])
        GUI.テーブル行追加(テーブル, ["鈴木一郎", "28", "名古屋"])
        初期化済 = 真
    終わり

    GUI.テーブル表示(テーブル)

    変数 選択行 = GUI.テーブル選択行(テーブル)
    もし 選択行 >= 0 なら
        GUI.ラベル("選択: 行 " + 選択行)
    終わり
終わり)

GUI.アプリ終了(アプリ)
```

> **💡 ポイント**: テーブルやツリーのような「保持される」データは、`初期化済` フラグを使って初回のみ作成します。描画ループ内で毎回作り直さないよう注意してください。

## 12. 描画 — 図形と色

キャンバスに自由に図形を描くことができます。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("描画", 600, 500)

GUI.描画ループ(アプリ, 関数():
    // 基本図形
    GUI.四角描画(50, 50, 200, 150, "#3498db")
    GUI.四角塗り(50, 50, 200, 150, "#3498db80")
    GUI.円描画(400, 125, 75, "#e74c3c")
    GUI.円塗り(400, 125, 75, "#e74c3c80")
    GUI.線描画(50, 250, 550, 250, "#2ecc71", 2)

    // 角丸四角
    GUI.角丸四角塗り(50, 280, 200, 100, 10, "#9b59b6")

    // テキスト描画
    GUI.テキスト描画("自由な位置にテキスト", 300, 350, "#ffffff")

    // グラデーション
    GUI.グラデーション四角(300, 280, 250, 100, "#e74c3c", "#3498db")
終わり)

GUI.アプリ終了(アプリ)
```

**描画関数：**

| 関数 | 説明 |
|---|---|
| `四角描画(x, y, w, h, 色)` | 四角形の枠線 |
| `四角塗り(x, y, w, h, 色)` | 塗りつぶし四角形 |
| `円描画(x, y, r, 色)` | 円の枠線 |
| `円塗り(x, y, r, 色)` | 塗りつぶし円 |
| `線描画(x1, y1, x2, y2, 色, 太さ)` | 直線 |
| `角丸四角塗り(x, y, w, h, r, 色)` | 角丸四角形 |
| `テキスト描画(テキスト, x, y, 色)` | 任意位置にテキスト |
| `画像描画(パス, x, y, w, h)` | 画像表示 |

> **💡 色の指定**: `"#RRGGBB"` 形式で指定します。末尾に透明度を追加できます（例: `"#FF000080"` で半透明赤）。

## 13. アニメーション

時間経過による自動的な値の変化を実現します。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("アニメーション", 600, 400)
変数 x座標 = 50
変数 アニメ中 = 偽

GUI.描画ループ(アプリ, 関数():
    もし GUI.ボタン("アニメーション開始") なら
        GUI.アニメーション開始("移動", 50, 500, 1.0)
        アニメ中 = 真
    終わり

    もし アニメ中 なら
        x座標 = GUI.アニメーション値("移動")
        もし GUI.アニメーション完了("移動") なら
            アニメ中 = 偽
        終わり
    終わり

    GUI.円塗り(x座標, 200, 30, "#e74c3c")
終わり)

GUI.アプリ終了(アプリ)
```

**アニメーション関数：**

| 関数 | 説明 |

## 19. 実践：TODOアプリを作る

これまで学んだ知識を組み合わせて、実用的な TODO アプリを作りましょう。

```
取り込む "hajimu_gui" として GUI

変数 アプリ = GUI.アプリ作成("TODOアプリ", 500, 600)

// 状態変数
変数 タスク一覧 = []
変数 完了一覧 = []
変数 新規タスク = ""
変数 フィルタ = 0

GUI.描画ループ(アプリ, 関数():
    GUI.見出し("📝 TODO リスト")

    // 新規タスク入力
    GUI.横並び開始()
        新規タスク = GUI.テキスト入力("新しいタスク", 新規タスク)
        もし GUI.ボタン("追加") なら
            もし 新規タスク != "" なら
                タスク一覧 = タスク一覧 + [新規タスク]
                完了一覧 = 完了一覧 + [偽]
                新規タスク = ""
            終わり
        終わり
    GUI.横並び終了()

    GUI.区切り線()

    // フィルタ
    フィルタ = GUI.ラジオボタン("表示", ["すべて", "未完了", "完了済み"], フィルタ)

    GUI.区切り線()

    // タスク一覧表示
    GUI.スクロール領域開始("タスク", 350)
        繰り返す 変数 i = 0 から タスク一覧の長さ - 1 まで
            // フィルタ適用
            変数 表示 = 偽
            もし フィルタ == 0 なら
                表示 = 真
            終わり
            もし フィルタ == 1 かつ 完了一覧[i] == 偽 なら
                表示 = 真
            終わり
            もし フィルタ == 2 かつ 完了一覧[i] == 真 なら
                表示 = 真
            終わり

            もし 表示 なら
                GUI.横並び開始()
                    完了一覧[i] = GUI.チェックボックス("##" + i, 完了一覧[i])
                    もし 完了一覧[i] なら
                        GUI.色付きラベル("✅ " + タスク一覧[i], "#888888")
                    でなければ
                        GUI.ラベル("⬜ " + タスク一覧[i])
                    終わり
                GUI.横並び終了()
            終わり
        終わり
    GUI.スクロール領域終了()

    // ステータス
    GUI.区切り線()
    変数 完了数 = 0
    繰り返す 変数 j = 0 から 完了一覧の長さ - 1 まで
        もし 完了一覧[j] == 真 なら
            完了数 = 完了数 + 1
        終わり
    終わり
    GUI.ラベル("完了: " + 完了数 + " / " + タスク一覧の長さ)
    GUI.プログレスバー("進捗", 完了数 / タスク一覧の長さ)
終わり)

GUI.アプリ終了(アプリ)
```

この TODO アプリでは以下の機能を使っています：

- **テキスト入力** + **ボタン** — タスクの追加
- **ラジオボタン** — フィルタリング
- **スクロール領域** — タスク一覧の表示
- **チェックボックス** — 完了状態の切替
- **色付きラベル** — 完了タスクの表示
- **プログレスバー** — 進捗の可視化

### よくある質問

**Q: 描画ループの中で毎回ウィジェットを呼ばないといけないの？**

A: はい。即時モードの GUI では、ウィジェットは毎フレーム呼ぶ必要があります。呼ばなければ画面に表示されません。

**Q: 状態はどこに持てばいい？**

A: 描画ループの**外**で `変数` として宣言してください。ループ内で宣言すると毎フレームリセットされます。

**Q: テーブルやツリーのデータは毎フレーム作り直す？**

A: いいえ。テーブルやツリーは保持モードです。`初期化済` フラグで初回のみ作成してください。

**Q: ウィンドウを複数作れる？**

A: `アプリ作成` を複数呼ぶことで複数ウィンドウを作成できます。
